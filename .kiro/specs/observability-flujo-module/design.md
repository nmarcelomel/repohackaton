# Diseño Técnico: Módulo de Observabilidad — Flujo ## Overview El módulo de Observabilidad para la página `/flujo` transforma la vista existente de tarjetas por equipo en un dashboard ejecutivo centrado en la célula seleccionada. Los usuarios (gerentes de ingeniería, líderes técnicos) seleccionan una célula de desarrollo y visualizan: métricas DORA (Deployment Frequency y Cycle Time) como tarjetas resumen, una barra WIP con indicadores de sobrecarga, tooltips educativos con texto empático, y un gráfico comparativo de impacto de IA en el Cycle Time. El diseño prioriza: - **Cero dependencias nuevas de charting** — se implementan barras comparativas con CSS/Tailwind puro - **Estado compartido ligero** — React Context API para el estado de equipo (isOverloaded) - **Extensibilidad** — interfaces tipadas que permiten conectar a servicios reales en el futuro - **Accesibilidad** — tooltips navegables por teclado, roles ARIA, contraste suficiente ## Architecture ### Diagrama de Componentes ```mermaid graph TD A[FlujoPage] --> B[CellSelector] A --> C[DoraMetricsCards] A --> D[WipAlertBar] A --> E[AIImpactChart] C --> F[MetricCard] F --> G[EducationalTooltip] D --> G A --> H[TeamStateProvider / Context] H --> I[Shared_Team_State] ``` ### Diagrama de Flujo de Datos ```mermaid sequenceDiagram participant U as Usuario participant CS as CellSelector participant FP as FlujoPage participant DS as Data_Service participant CTX as TeamStateContext FP->>DS: getTeams() DS-->>FP: Team[] FP->>FP: setSelectedTeam(teams[0]) FP->>CS: teams, selectedTeamId FP->>DS: getObservabilityMetrics(teamId) DS-->>FP: ObservabilityMetrics FP->>CTX: updateTeamOverloadState(teamId, isOverloaded) U->>CS: Selecciona otra célula CS->>FP: onTeamChange(teamId) FP->>DS: getObservabilityMetrics(teamId) DS-->>FP: ObservabilityMetrics (nueva célula) FP->>CTX: updateTeamOverloadState(teamId, isOverloaded) ``` ### Capas del Sistema | Capa | Responsabilidad | Archivos | |------|----------------|----------| | Presentación | Renderizado de componentes, interacción de usuario | `src/pages/flujo/components/*.tsx` | | Estado | Context API para estado compartido entre módulos | `src/pages/flujo/context/TeamStateContext.tsx` | | Datos | Abstracción de acceso a datos mock | `src/data/data-service.ts` (extensión) | | Tipos | Interfaces TypeScript estrictas | `src/types/index.ts` (extensión) |
